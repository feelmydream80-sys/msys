# 사용자 요구사항 분석 및 실행 계획

## 문서 정보
- **작성일**: 2025-12-23
- **버전**: 1.0
- **분석 대상**: 사용자의 2가지 구체적 UI 개선 요구사항
- **목적**: 요구사항 분석, 구현 계획 수립, 체크리스트 기반 실행

## 1. 요구사항 분석

### 1.1. 요구사항 1: 카드 요약 표시 옵션 라디오 박스 위치 조정
**원문**: "카드 요약 표시 옵션의 라디오 박스는 타이틀 우측 상단에 이동시켜서 높이를 줄여줘."

**분석**:
- **현재 상태**: `card_summary.html`에서 라디오 박스들이 카드 콘텐츠 영역 하단에 위치
- **요구사항**: 라디오 박스들을 카드 헤더의 우측 상단으로 이동
- **목적**: 카드 높이를 줄이고, 옵션 선택을 더 직관적으로 배치
- **영향 범위**: `templates/card_summary.html` 수정 필요

**구현 세부사항**:
- 카드 헤더에 라디오 박스들을 배치
- flex 레이아웃으로 타이틀(좌측)과 라디오 박스(우측) 배치
- 카드 콘텐츠 영역 제거 또는 최소화로 높이 축소

### 1.2. 요구사항 2: 수집 요청서 양식 다운로드 기능 추가
**원문**: "collection_schedule의 수집 요청서 양식 다운로드과 같은 형태로 card_summary에 추가해줘."

**분석**:
- **참조 대상**: `templates/collection_schedule.html`의 다운로드 버튼
- **현재 구현**: collection_schedule.html에 녹색 다운로드 버튼 존재
- **요구사항**: 동일한 스타일과 위치의 다운로드 기능을 card_summary.html에 추가
- **목적**: card_summary 메뉴에도 엑셀 템플릿 다운로드 기능 제공
- **영향 범위**: `templates/card_summary.html`에 버튼 추가, 관련 기능 구현

**구현 세부사항**:
- collection_schedule.html의 버튼 스타일 복사
- card_summary.html의 적절한 위치에 배치 (헤더 우측 또는 별도 영역)
- 엑셀 템플릿 다운로드 기능 구현 (기존 API 활용 또는 신규 구현)

## 2. 구현 계획

### 2.1. 작업 순서
1. **요구사항 1 구현**: 라디오 박스 위치 조정
2. **요구사항 2 구현**: 다운로드 버튼 추가
3. **테스트 및 검증**: UI 및 기능 테스트

### 2.2. 예상 파일 변경
- `templates/card_summary.html`: 메인 수정 대상
- `static/css/card_summary.css`: 스타일 조정 (필요시)
- 기타: 다운로드 기능 관련 파일들

### 2.3. 위험 요소
- **UI 레이아웃 깨짐**: flex 레이아웃 변경으로 인한 표시 문제
- **기능 충돌**: 다운로드 버튼 추가로 인한 기존 기능 영향
- **반응형 디자인**: 모바일 환경에서의 표시 문제
